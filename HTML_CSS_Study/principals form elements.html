Principals Atributtes

accept-charset: define which charset will be used by the form. press control+U to see HTML source code:

autocomplete: when we define the autocomplete attribute as we use the fields in a form the browser will start to suggest values for what we are typing, based on values that were typed previously. press control+U to see HTML source code:

action: specifies where the form data will be sent. It will usually be a page link on the server

method: Specifies the HTTP method that will be used for the submission (get or post):

enctype(only works for POST type methods): Specifies when form data needs to be encoded before it is sent to the server.

required: used when informing the browser that an element of the form is mandatory. exclusive use with input, select and textarea elements.

type=”email”: no input means setting the field to receive a valid email address. (does not check if the email entered exists), but if the format is correct.

pattern: Using the 'pattern' attribute you can require the user to enter values in a pattern made with regular expression, that is, what is typed in the compo input must satisfy the regular expression created by the developer.
(xx) xxxx-xxxx

novalidate: specifies that form elements should not be validated when submitted even if they have 'required', type with 'email', 'url', etc.

target: As with the '< a >' tag, the target indicates where the target page will open.
